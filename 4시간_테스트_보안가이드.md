## 0. tools 컨테이너 접속 (테스트 실행 환경)

테스트·보안 명령은 **`tools` 컨테이너 안에서 실행**하면 호스트 OS(Windows/Mac/Linux)에 관계없이
동일하게 동작합니다(curl·jq·python3·nmap 내장). 호스트엔 **Docker 만** 있으면 됩니다.

### 0-1. 컨테이너 확인 / 기동
```bash
cd ai-siem-lab/labs/opensearch-ai
docker compose ps tools          # STATUS = Up 확인
docker compose up -d tools       # 안 떠 있으면 기동
```

### 0-2. 대화형 셸로 접속 (권장)
```bash
docker compose exec tools bash
#  → 프롬프트가 컨테이너 내부로 전환됨. 작업 디렉터리는 /lab (랩 폴더 마운트)
#  이후 명령은 localhost 대신 서비스명 사용:
curl -s http://opensearch:9200/_cluster/health | jq .status
exit                              # 컨테이너에서 빠져나오기
```
> `docker compose` 컨텍스트 밖이면 직접 접속도 가능: `docker exec -it ai-siem-tools bash`

### 0-3. 단발 명령 (셸 진입 없이)
```bash
docker compose exec tools curl -s http://opensearch:9200/security-web/_count | jq .count
docker compose exec tools python3 sample-data/load_sample_logs.py http://opensearch:9200
docker compose exec tools nmap -p 9200,5601,11434 opensearch ollama
```

### 접속 후 핵심
| 항목 | 값 |
|------|-----|
| 내부 주소 | OpenSearch `http://opensearch:9200` · Ollama `http://ollama:11434` (※ `localhost` 아님) |
| 작업 폴더 | `/lab` (= `labs/opensearch-ai`, `sample-data/` 포함) |
| 내장 도구 | curl · jq · python3 · nmap · dig · nc |

> 이후 `curl`/`python3`/`nmap` 같은 API·도구 명령은 컨테이너에서 실행합니다(주소 `opensearch:9200`/`ollama:11434`).
> `docker compose …` 관리 명령은 **호스트**에서 실행하세요. 호스트에서 API 직접 실행 시 서비스명→`localhost` 로 바꿉니다.

## 1. 목적

탐지 → 분석 → 보고서까지 **전 과정(end-to-end)** 이 끊김 없이 동작하는지 검증하고, 종합 공격
시나리오로 관제 흐름 전체를 보안 관점에서 점검합니다. 모델 ID 준비(품질 모델 우선):
```bash
MID=$(curl -s -H 'Content-Type: application/json' http://opensearch:9200/_plugins/_ml/models/_search \
  -d '{"size":20,"query":{"bool":{"must":[{"term":{"model_state":"DEPLOYED"}},{"term":{"algorithm":"REMOTE"}}]}}}' \
  | jq -r '[.hits.hits[]|{id:._id,t:(._source.name+" "+(._source.description//""))}]
           | (map(select(.t|test("8b|7b|3b";"i")))[0].id) // .[0].id')
echo "MODEL_ID=$MID  ($(curl -s http://opensearch:9200/_plugins/_ml/models/$MID | jq -r .name))"
```

## 2. 검증(테스트) 가이드 — End-to-End

### 2-1. 탐지: 공격 IP 식별
```bash
curl -s -H 'Content-Type: application/json' http://opensearch:9200/security-web/_search -d '{
  "size":0,"aggs":{"by_ip":{"terms":{"field":"source.ip","size":3},
  "aggs":{"errs":{"filter":{"range":{"http.response.status_code":{"gte":400}}}}}}} }' \
| jq '.aggregations.by_ip.buckets[] | {ip:.key, total:.doc_count, errors:.errs.doc_count}'
```
**기대**: 공격 IP(10.13.37.7, 45.155.205.99)가 오류 비율 상위.

### 2-2. 상세 조사
```bash
curl -s -H 'Content-Type: application/json' http://opensearch:9200/security-web/_search -d '{
  "size":3,"query":{"term":{"source.ip":"45.155.205.99"}},
  "_source":["url.path","url.query","http.response.status_code"] }' | jq '.hits.hits[]._source'
```
**기대**: SQLi 페이로드(UNION SELECT) 로그 반환.

### 2-3. LLM 분석 → 보고서 초안
```bash
curl -s -H 'Content-Type: application/json' http://opensearch:9200/_plugins/_ml/models/$MID/_predict -d '{
  "parameters":{"messages":[
    {"role":"system","content":"너는 SOC 분석가다. 관제 보고서 초안을 한국어로: 개요/탐지/영향/권고."},
    {"role":"user","content":"45.155.205.99 가 /api/items 에 sqlmap UA로 UNION SELECT 다수 전송, 일부 500."}]}}' \
| jq -r '.inference_results[0].output[0].dataAsMap.choices[0].message.content'
```
**기대**: 개요/탐지/영향/권고 구조의 보고서 초안 생성.

## 3. 보안 테스트 (종합 시나리오)

### 3-1. 멀티 단계 공격 재현 → 탐지 커버리지
정찰(스캐닝) → 익스플로잇(SQLi) 2단계를 주입하고 모두 탐지되는지 확인:
```bash
ts=$(date -u +%Y-%m-%dT%H:%M:%SZ)
( # 정찰: 404 다수
  for p in /admin /backup /.env /wp-login.php /phpmyadmin; do
    echo '{"index":{"_index":"security-web"}}'
    echo "{\"@timestamp\":\"$ts\",\"source\":{\"ip\":\"198.51.100.7\"},\"url\":{\"path\":\"$p\"},\"http\":{\"response\":{\"status_code\":404}},\"user_agent\":{\"original\":\"nikto\"},\"event\":{\"category\":\"web\"}}"
  done
  # 익스플로잇: SQLi
  echo '{"index":{"_index":"security-web"}}'
  echo "{\"@timestamp\":\"$ts\",\"source\":{\"ip\":\"198.51.100.7\"},\"url\":{\"path\":\"/api/items\",\"query\":\"id=1 UNION SELECT password FROM users\"},\"http\":{\"response\":{\"status_code\":500}},\"user_agent\":{\"original\":\"sqlmap/1.7\"},\"event\":{\"category\":\"web\"}}"
) | curl -s -H 'Content-Type: application/x-ndjson' 'http://opensearch:9200/_bulk?refresh=true' --data-binary @- >/dev/null

# 탐지 1) 정찰(404 다수)  2) SQLi
curl -s -H 'Content-Type: application/json' http://opensearch:9200/security-web/_search -d '{
 "size":0,"query":{"term":{"source.ip":"198.51.100.7"}},
 "aggs":{"by_status":{"terms":{"field":"http.response.status_code"}}} }' | jq '.aggregations.by_status.buckets'
```
**기대**: `198.51.100.7` 에서 404 다수 + 500(SQLi) 동시 관측 → **다단계 공격 탐지**.
(정리: 샘플 로더 재실행 시 초기화)

### 3-2. 대응 검증 (개념 + 확인)
- 차단 정책 수립 후 동일 IP 재시도가 차단되는지(운영: WAF/방화벽). 본 실습은 탐지까지이며,
  대응은 SOAR(n8n/Shuffle) 연계로 자동화 → 가이드의 "운영 전환" 참고.

### 3-3. 보고서 정확성 검증
2-3 산출 보고서에서 다음을 점검:
- [ ] 공격 IP·유형이 **사실과 일치**(환각 없음)
- [ ] 심각도·권고가 합리적
- [ ] 민감정보 과다 노출 없음
→ **사람 검수 필수**(LLM 초안은 보조).

### 3-4. 운영 보안 체크리스트 (종합)
- [ ] 보안 플러그인·TLS·RBAC 활성(운영)
- [ ] LLM 프롬프트 인젝션 방어(3시간 가이드)
- [ ] 데이터 경계: 로컬 LLM·허용 엔드포인트만
- [ ] 탐지 커버리지(ATT&CK) + 오탐 관리
- [ ] 감사 로깅·백업·접근 통제
