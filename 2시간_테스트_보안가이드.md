## 0. tools 컨테이너 접속 (테스트 실행 환경)

테스트·보안 명령은 **`tools` 컨테이너 안에서 실행**하면 호스트 OS(Windows/Mac/Linux)에 관계없이
동일하게 동작합니다(curl·jq·python3·nmap 내장). 호스트엔 **Docker 만** 있으면 됩니다.

### 0-1. 컨테이너 확인 / 기동
```bash
cd ai-siem-lab/labs/opensearch-ai
docker compose ps tools          # STATUS = Up 확인
docker compose up -d tools       # 안 떠 있으면 기동
```

### 0-2. 대화형 셸로 접속 (권장)
```bash
docker compose exec tools bash
#  → 프롬프트가 컨테이너 내부로 전환됨. 작업 디렉터리는 /lab (랩 폴더 마운트)
#  이후 명령은 localhost 대신 서비스명 사용:
curl -s http://opensearch:9200/_cluster/health | jq .status
exit                              # 컨테이너에서 빠져나오기
```
> `docker compose` 컨텍스트 밖이면 직접 접속도 가능: `docker exec -it ai-siem-tools bash`

### 0-3. 단발 명령 (셸 진입 없이)
```bash
docker compose exec tools curl -s http://opensearch:9200/security-web/_count | jq .count
docker compose exec tools python3 sample-data/load_sample_logs.py http://opensearch:9200
docker compose exec tools nmap -p 9200,5601,11434 opensearch ollama
```

### 접속 후 핵심
| 항목 | 값 |
|------|-----|
| 내부 주소 | OpenSearch `http://opensearch:9200` · Ollama `http://ollama:11434` (※ `localhost` 아님) |
| 작업 폴더 | `/lab` (= `labs/opensearch-ai`, `sample-data/` 포함) |
| 내장 도구 | curl · jq · python3 · nmap · dig · nc |

> 이후 `curl`/`python3`/`nmap` 같은 API·도구 명령은 컨테이너에서 실행합니다(주소 `opensearch:9200`/`ollama:11434`).
> `docker compose …` 관리 명령은 **호스트**에서 실행하세요. 호스트에서 API 직접 실행 시 서비스명→`localhost` 로 바꿉니다.

## 1. 목적

탐지 쿼리와 이상 탐지가 **실제 공격을 잡아내는지** 검증하고, **오탐(정상 트래픽 미탐지)** 을
확인합니다. 보안 테스트의 핵심은 *positive(공격은 잡힌다)* + *negative(정상은 안 잡힌다)* +
*replay(새 공격도 잡힌다)* 의 3축 검증입니다.

---

## 2. 검증(테스트) 가이드

### 2-1. 브루트포스 탐지 (검증값 일치)
```bash
curl -s -H 'Content-Type: application/json' http://opensearch:9200/security-web/_search -d '{
  "size":0, "query":{"term":{"http.response.status_code":401}},
  "aggs":{"by_ip":{"terms":{"field":"source.ip","size":5}}} }' \
| jq '.aggregations.by_ip.buckets[] | {ip:.key, c:.doc_count}'
```
**기대**: `10.13.37.7` 의 `c=120` (최상위).

### 2-2. SQLi 탐지
```bash
curl -s -H 'Content-Type: application/json' http://opensearch:9200/security-web/_search -d '{
  "size":5, "_source":["source.ip","user_agent.original"],
  "query":{"wildcard":{"url.query":"*UNION*"}} }' | jq '.hits.hits[]._source'
```
**기대**: `45.155.205.99` / `sqlmap/1.7`.

### 2-3. Anomaly Detector 상태
```bash
curl -s -H 'Content-Type: application/json' http://opensearch:9200/_plugins/_anomaly_detection/detectors/_search \
  -d '{"query":{"match":{"name":"web-ip-spike"}}}' | jq '.hits.total.value'
```
**기대**: `1` 이상 (생성됨).

## 3. 보안 테스트 (탐지 검증)

### 3-1. Positive — 알려진 공격이 잡히는가
2-1·2-2 가 공격 IP를 반환하면 통과. ATT&CK 매핑:

| 공격 | 탐지 단서 | ATT&CK |
|------|-----------|--------|
| 브루트포스 | 401 급증 / `python-requests` | T1110 |
| SQLi | `UNION SELECT` / `sqlmap` | T1190 |
| XSS·스캐닝 | `<script>` / 다수 404 | T1059 / T1595 |

### 3-2. Negative — 정상만 있을 때 오탐 없는가
```bash
# 정상망(192.168.10.0/24) 중 401 이 거의 없어야 함
# source.ip 는 ip 타입 → prefix 불가, CIDR 를 term 으로 매칭
curl -s -H 'Content-Type: application/json' http://opensearch:9200/security-web/_search -d '{
  "size":0,"query":{"bool":{"must":[{"term":{"source.ip":"192.168.10.0/24"}},
  {"term":{"http.response.status_code":401}}]}} }' | jq '.hits.total.value'
```
**기대**: `0` 또는 매우 작음 → 정상 트래픽은 브루트포스로 오탐되지 않음.

### 3-3. Replay — 새 공격을 주입해 탐지되는지 (공격 시뮬레이션)
새 공격자 IP `203.0.113.200` 의 401 버스트 30건을 주입한 뒤 탐지 쿼리 재실행:
```bash
ts=$(date -u +%Y-%m-%dT%H:%M:%SZ)
( for i in $(seq 1 30); do
    echo '{"index":{"_index":"security-web"}}'
    echo "{\"@timestamp\":\"$ts\",\"source\":{\"ip\":\"203.0.113.200\"},\"url\":{\"path\":\"/login\"},\"http\":{\"request\":{\"method\":\"POST\"},\"response\":{\"status_code\":401}},\"user_agent\":{\"original\":\"hydra\"},\"event\":{\"category\":\"authentication\"}}"
  done ) | curl -s -H 'Content-Type: application/x-ndjson' \
    'http://opensearch:9200/_bulk?refresh=true' --data-binary @- > /dev/null
# 재탐지
curl -s -H 'Content-Type: application/json' http://opensearch:9200/security-web/_search -d '{
  "size":0,"query":{"term":{"http.response.status_code":401}},
  "aggs":{"by_ip":{"terms":{"field":"source.ip","size":5}}} }' \
| jq '.aggregations.by_ip.buckets[] | {ip:.key, c:.doc_count}'
```
**기대**: `203.0.113.200` 이 새로 상위에 등장 → **탐지 룰이 신규 공격도 포착**.
(정리: `python .../sample-data/load_sample_logs.py` 재실행 시 인덱스 초기화)

### 3-4. 탐지 갭/오탐 점검
- **갭**: 난독화된 SQLi(인코딩), 분산 브루트포스(IP 분산)는 단순 룰을 우회 → 이상탐지로 보완
- **오탐**: 정상 급증(이벤트·배치)이 이상탐지에 걸릴 수 있음 → 임계값·화이트리스트 조정

## 4. 통과 기준

| 테스트 | 통과 조건 |
|--------|-----------|
| Positive | 브루트포스/SQLi IP 정확히 반환 |
| Negative | 정상 IP 401 ≈ 0 |
| Replay | 주입한 신규 공격 IP 탐지 |
| 인지 | 탐지 갭·오탐 사유 설명 가능 |
